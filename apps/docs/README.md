# Documentation site (content)

Long-form documentation **content** lives here for architecture and product narrative.

The **in-app documentation** experience is implemented in `apps/dashboard` (`DocumentationPage` and related routes). This directory is the home for:

- `content/` — Markdown or MDX sources (optional static site generator later)
- Deep dives that also appear in `/docs/*.md` at the repo root

For judge-facing API and SDK docs, see root **`docs/`** (`api-reference.md`, `sdk-guide.md`).
